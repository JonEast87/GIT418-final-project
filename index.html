Tutorial - Intermediate JavaScript Functions

Author: Jonathan Eastin
10:00PM Dec. 2nd

I want to talk a little about intermediate and advanced functions built into the JavaScript language. Most students are aware of the more rudimentary loops such as, "for", "do while" and "while".

However there exists more than just those, and in several cases they are superior at certain tasks than the others.

I am going to begin with the filter function call. Say I wanted to only have only numbers less than five taken from one array and put into another?

Well you would probably start that solution off like:

      
        let lessThanFive = new Array()
        for (let i = 0; i < array.length; i++) {
          if (array[i] < 5) {
            lessThanFive.push(array[i])
          }
        }

Seems simple right? Well there is a better and quicker way to accomplish.

      
        lessThanFive = numbers.filter(number => number < 5)

You might be wondering, "That's it?", "No setting up the array length? No instantiate of i or any other vowel to track the loop's progress?"

Nope none of that. In fact the filter method can be used in many ways not just for sifting numbers, but word lengths and element types.

Let's look at another intermediate function call in JavaScript. My personal favorite, "map".

If you wanted to double the values of each array's elements you could do something similar to this:

      
        let array = [1, 2, 3, 4]
        let doubled = new Array()
        for (let i = 0; i < array.length; i++) {
          doubled.push(2 * array[i])
        }

Instead with map we can do that in just two lines:

      
        let array = [1, 2, 3, 4]
        let doubled = array.map(x => x * 2)

It's magnificent isn't it? Now while these solutions are not anymore efficent than the others in terms of time execution they do safe lines and make your code much more manageable.

Also to note, is that since they are technically functions they can also be tied to const and let variables for ease of or reuse, whereas having to do the same for a stand for loop would require more typing of the function shell it would inhabit. With these intermediate functions that is already built it.

If you would like to play around with these code snippets a bit or learn more about them their jsfiddle links will be below:

You, me and HTTPS

Author: Jonathan Eastin
9:00PM Nov. 19th

Most of the denizens of the web are not necessarily familiar with a websites "protocol", which has for years been 'http'. This standard however is being replaced with a new type of protocol 'https'.

Upon loading up a website both the website and your browser will communicate and exchange the necessary information needed to load the page you requested. This exchange is called a 'protocol' and it has been headed by http for years.

With the introduction of Https this transfer has become more secure (the S in Https standing for 'secure'). What happens now is that when your browser communicates with the website, the website will pass your broswer a SSL certificate that is used for that one time to create a secure connection between the two.

This does not mean however that the website you are accessing will not damage or intrude your system. It only means that the transfer of information between your computer and the site is secure.[1]

Dieting on Cookies

Author: Jonathan Eastin
9:00PM Nov. 19th

You are probably wondering just why would someone diet on cookies only? Well on the internet websites generally save small text files onto your computer, these text files are refered to as "cookies".

While cookies are mostly innocuous to you and your system since they mostly store information like username/password to your computer, they can be used for other purposes as well.

Cookies can also include other miscellanous information that the site may require of you.

One of the other uses of cookies and prehaps the most notorious is their abiliy to track you. These can in cases like 'Google Adsense' track you across multiple unrelated sites and target ad you with relelvant or recently searched for items or services.

It is important for users to periodically purge their cookie folders ever so often via their browser or operating system. Browsers have streamlined this process to even allow users to flush the cookie folder upon closing of the browser.[2]

JavaScript? JavaScript?!

Author: Jonathan Eastin
9:00PM Nov. 19th

An incredibly popular programming language that began as a web specific language but has now begun to find itself in more non web applications through various mediums (electron and NodeJS, amongst others).

Because of it's popularity it is important for web developers to know of it, to be aware of it and also what it is capable of. The popularity of the language also makes it a frequent vehicle of attack on websites and JavaScript based applications.

Some of the most common of these are cross-site scripting (XSS) and server-side JavaScript injection.

XSS has been around for sometime compared to server side injection. XSS involves injecting malicious code (often times by the owner themselves unknowningly) into a vulnerable application and using that application as a vehicle to spread the code or to steal important information from your applications users'.

It is best avoided by not injecting unknown scripts into your webpage.

Serer side injection occurs when a user injects malicious code into your site. Not to be confused with XSS which is the owner themselves accidentally (or purposely) injecting the malicious code into their own website. Users this time are the ones possibly compromising websites.

A major injection exploit was discovered when a WordPress plugin was found to have been vulnerable to SSI, which affected over 400,000 websites. Users can carry out these attacks this in several ways.

Some of the easiest ways to avoid SSI is to avoid the use of JavaScript serialization in your html input fields, avoid the use of eval(), setTimeout(), setInterval() and new Function() in your programs. NodeJS security linters are also being created to further help developers to prevent SSI.[3]
